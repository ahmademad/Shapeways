    Adventure Question


    Describe the MVC design pattern as it applies to web servers - start from initial HTTP request and end with writing the response.
    1. Request comes in (e.g. GET resource A) and is fielded by a DispatcherServlet.
    2. DispatcherServlet dispatches the request to the appropriate controller.
    3. The controller interprets the URI's query parameters and queries the model.
    4. The model contains the domain data and this is received and formatted ready for viewing
    5. The controller returns typically a view name which the Dispatcher looks up to get an actual view (e.g. a JSP)
    6. The Dispatcher passes the formatted model data to the view (JSP)
    7. The view renders the model data in HTML
    8. The HTML is sent back to the client and rendered in the browser
    This structure allows views and models to remain as independent as possible, which promotes re-use and increases the code's flexibility.

    Describe the pros and cons of version control.
    • + allows concurrent modification of source files
    • + allows feature branch creation and merging
    • + helps prevent catastrophic loss of source code
    • + helps to foster a professional attitude toward coding
    • + allows you to "go back in time" to any previous version of the code (including renames in git)
    • + encourages comments to be made for each change
    • + performs a lot of merging work for you as opposed to merging by hand
    • + great way of checking that no-one's modified deployed production code (put it in a repository)
    • - can be steep learning curve e.g. git
    • - when learning, people can trash the repository and delay colleagues
    • - when making lots of quick changes, having to do a checkin for each one can feel onerous
    • - some edge cases can be time-consuming e.g. dealing with binary files, CRLF v.s. LF, file permissions etc.
